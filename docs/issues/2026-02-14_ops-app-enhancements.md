# Approach

- **アイテム新規追加**: KvTable に「追加」ボタンを設置し、押下時にJSON入力用のフォーム（テキストエリア）を表示する。編集モードと同様のUIを流用し、POSTリクエストでバックエンドにデータを送信する。ルートハンドラにPOSTメソッドを追加し、KVストアへの書き込みを行う。`id` は自動生成、`createdAt`/`updatedAt` はサーバー側で付与する。
- **アイテム検索**: KvTable 上部に検索入力欄を追加し、クライアントサイドでフィルタリングを行う。現在取得済みのデータに対してキーまたはバリュー内の文字列で絞り込む。件数が多い場合はサーバーサイド検索も検討するが、まずはクライアントサイドで対応する。
- **追加ロード**: 現段階では見送り。実装する場合は、KV list のカーソルを利用したページネーションを導入し、「Load more」ボタンで次のページを取得・既存リストに追加する形にする。

# Results

- **並び順変更**: 実装済み。`kv.list()` に `reverse: true` を指定し、新しいアイテムが上に表示されるようにした。
- **削除モード**: 実装済み。ツールバーにゴミ箱アイコンボタンを追加。削除モード切替でチェックボックスが表示され、選択アイテムの一括削除が可能。確認ダイアログ付き。
- **編集モード**: 実装済み。各行に編集（ペンシル）アイコンボタンを配置。JSON形式のテキストエリアで値を編集でき、保護フィールド（id, createdAt, updatedAt）の変更はサーバー側で拒否される。
