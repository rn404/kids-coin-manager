Start working on the issue:

1. Read `/docs/issues/_Rules.md` to understand the status definitions and document format
2. Read the issue document from `/docs/issues/$ARGUMENTS`
3. Review the current status and contents
4. If "Approach" section is empty or incomplete:
   - Discuss and fill in the concrete work plan
   - Update Status according to the rules in `_Rules.md`
5. Before making code changes:
   - Confirm the approach with the user
6. During work:
   - Update the issue document sections as work progresses
   - If research is needed, save reports to `/docs/reports/` and link in References
7. Periodically update the issue document with progress

Remember: The issue document is the source of truth for this work session.
