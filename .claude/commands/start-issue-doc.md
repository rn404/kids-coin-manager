argument-hint: "<issue-file-name>"
---

Start working on the issue:

1. Read the issue document from `/docs/issues/$ARGUMENTS`
2. Review the current status and approach
3. If "Approach" section is empty or incomplete:
   - Discuss and fill in the concrete work plan
   - Update the issue document
4. Before making code changes:
   - Add today's entry to Work Log section
   - Document the planned changes
5. During work:
   - Update Work Log with progress and findings
   - If research is needed, save reports to `/docs/reports/` and link in References
6. Periodically update the issue document with progress

Remember: The issue document is the source of truth for this work session.
